[OpenClaw](https://openclaw.ai) is building an open ecosystem for AI agent skills -- a place where developers publish, discover, and install capabilities for agents like Claude Code, Cursor, and Windsurf. Panguard Skill Auditor is the security infrastructure that makes this ecosystem trustworthy.

## Why OpenClaw Needs a Security Layer

Mobile app stores learned this lesson years ago: an open platform without security review becomes a malware distribution channel. The same principle applies to AI skill ecosystems.

| App Store | OpenClaw Ecosystem |
|-----------|-------------------|
| App review process | **Panguard Skill Auditor** |
| Malware scanning | Prompt injection + tool poisoning detection |
| Permission review | Permission scope analysis |
| Code signing | Manifest validation + integrity checks |
| User ratings | Quantitative risk score (0-100) |

Without automated security scanning, every skill installation is a trust decision made blindly. Panguard makes that decision informed.

## Integration Points

### 1. npm Package Integration (Recommended)

The fastest way to add Skill Auditor to any Node.js project. No CLI installation, no shell scripts -- just `npm install` and import.

```bash
npm install @panguard-ai/panguard-skill-auditor
```

#### Basic Usage

```typescript
import { auditSkill } from '@panguard-ai/panguard-skill-auditor';

const report = await auditSkill('./skills/some-community-skill');

console.log(report.riskScore);   // 0-100
console.log(report.riskLevel);   // "LOW" | "MEDIUM" | "HIGH" | "CRITICAL"
console.log(report.verdict);     // Human-readable verdict string
console.log(report.findings);    // AuditFinding[] with severity, message, line numbers
```

#### Pre-Install Gate in TypeScript

```typescript
import { auditSkill } from '@panguard-ai/panguard-skill-auditor';

async function safeInstall(skillPath: string): Promise<boolean> {
  const report = await auditSkill(skillPath);

  if (report.riskLevel === 'CRITICAL') {
    console.error(`BLOCKED: ${skillPath} has critical issues`);
    report.findings
      .filter(f => f.severity === 'CRITICAL')
      .forEach(f => console.error(`  [${f.severity}] ${f.message} (line ${f.line})`));
    return false;
  }

  if (report.riskLevel === 'HIGH') {
    console.warn(`WARNING: ${skillPath} scored ${report.riskScore}/100`);
    report.findings.forEach(f => console.warn(`  [${f.severity}] ${f.message}`));
    // Require manual approval for HIGH risk
    return false;
  }

  // LOW or MEDIUM -- safe to install
  return true;
}
```

#### Custom AI Provider (Optional)

Layer 2 semantic analysis uses an LLM. You can provide your own:

```typescript
import { auditSkill } from '@panguard-ai/panguard-skill-auditor';
import type { SkillAnalysisLLM } from '@panguard-ai/panguard-skill-auditor';

const myLLM: SkillAnalysisLLM = {
  analyze: async (prompt: string) => {
    // Call your preferred LLM (Claude, GPT, local model, etc.)
    const response = await fetch('https://api.anthropic.com/v1/messages', {
      method: 'POST',
      headers: { 'x-api-key': process.env.ANTHROPIC_API_KEY!, 'content-type': 'application/json', 'anthropic-version': '2023-06-01' },
      body: JSON.stringify({ model: 'claude-haiku-4-5-20251001', max_tokens: 1024, messages: [{ role: 'user', content: prompt }] }),
    });
    const data = await response.json();
    return data.content[0].text;
  },
};

const report = await auditSkill('./skills/my-skill', { llm: myLLM });
```

<Info>
  Without a custom LLM, Layer 2 (AI semantic analysis) is skipped. Layer 1 regex checks still run and catch the majority of threats.
</Info>

### 2. Pre-Install Gate (CLI)

The simplest integration: run Skill Auditor before every `claw install`.

```bash
# Audit before install
panguard audit skill ./skills/new-skill --json | jq '.riskLevel'

# Automated gate
audit_and_install() {
  local SKILL_PATH="$1"
  local RESULT=$(panguard audit skill "$SKILL_PATH" --json)
  local RISK=$(echo "$RESULT" | jq -r '.riskLevel')
  local SCORE=$(echo "$RESULT" | jq -r '.riskScore')

  echo "Risk: $RISK ($SCORE/100)"

  if [ "$RISK" = "CRITICAL" ]; then
    echo "BLOCKED: Critical security issues found"
    echo "$RESULT" | jq '.findings[]'
    return 1
  fi

  if [ "$RISK" = "HIGH" ]; then
    echo "WARNING: High risk. Review findings before installing."
    echo "$RESULT" | jq '.findings[]'
    read -p "Continue anyway? (y/N) " confirm
    [ "$confirm" = "y" ] || return 1
  fi

  claw install "$SKILL_PATH"
}
```

### 3. CI Pipeline for Skill Repositories

Add Skill Auditor to your GitHub Actions workflow to scan skills on every PR. You can use either the npm package or the CLI.

<Tabs>
  <Tab title="npm (Recommended)">
```yaml
# .github/workflows/skill-audit.yml
name: Skill Security Audit
on:
  pull_request:
    paths:
      - 'skills/**'

jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Skill Auditor
        run: npm install @panguard-ai/panguard-skill-auditor

      - name: Audit all skills
        run: |
          node -e "
            const { auditSkill } = require('@panguard-ai/panguard-skill-auditor');
            const fs = require('fs');
            const path = require('path');
            (async () => {
              const dirs = fs.readdirSync('skills', { withFileTypes: true })
                .filter(d => d.isDirectory())
                .map(d => path.join('skills', d.name));
              let failed = false;
              for (const dir of dirs) {
                const report = await auditSkill(dir);
                console.log(dir + ': ' + report.riskLevel + ' (' + report.riskScore + '/100)');
                if (report.riskLevel === 'CRITICAL' || report.riskLevel === 'HIGH') {
                  report.findings.forEach(f => console.error('  [' + f.severity + '] ' + f.message));
                  failed = true;
                }
              }
              if (failed) process.exit(1);
            })();
          "
```
  </Tab>
  <Tab title="CLI">
```yaml
# .github/workflows/skill-audit.yml
name: Skill Security Audit
on:

jobs:

              echo "$RESULT" | jq '.findings[]'
              exit 1
            fi
          done

```

## The Trust Chain

Panguard creates a verifiable trust chain for every skill in the OpenClaw ecosystem:

<Steps>
  <Step title="Publish">
    Developer publishes a skill to the OpenClaw registry with a SKILL.md manifest.
  </Step>
  <Step title="Scan">
    Panguard Skill Auditor runs 7 checks and assigns a risk score.
  </Step>
  <Step title="Record">
    The audit result is stored locally. Registry-level attestation is on the roadmap.
  </Step>
  <Step title="Enforce">
    Organizations set policies that block skills above their risk threshold.
  </Step>
  <Step title="Monitor">
    Skills are re-audited when updated. Score changes trigger alerts via Panguard Chat.
  </Step>
</Steps>

## Supported Skill Formats

Skill Auditor supports any skill that follows the OpenClaw SKILL.md specification:

| Format | Support |
|--------|---------|
| OpenClaw SKILL.md | Full support |
| Claude Code custom commands | Full support |
| Cursor rules files | Partial (manifest checks skipped) |
| Generic markdown skill files | Partial (content checks only) |
